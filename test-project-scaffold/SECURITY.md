## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest | Active support |
| Previous | Security fixes only |
| Older | No longer supported |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Report privately via one of these channels:

- **Email**: security@example.com
- **GitHub Security Advisories**: {{FILL_ME:REPO_URL}}/security/advisories

### What to Include

- Description of the vulnerability and its potential impact
- Affected versions and configurations
- Step-by-step reproduction instructions
- Proof-of-concept code (if available)
- Your suggested fix (optional but appreciated)

## Response Timeline

| Stage | Timeline |
|-------|----------|
| Acknowledgment | Within 48 hours |
| Initial assessment | Within 5 business days |
| Fix development | Within 30 days (critical) / 90 days (moderate) |
| Public disclosure | After fix is released and 7 days have passed |

## Disclosure Policy

We follow coordinated disclosure. We will:

1. Acknowledge your report promptly
2. Keep you informed of progress
3. Credit you in the security advisory (unless you prefer anonymity)
4. Not take legal action against good-faith security researchers

## Security Best Practices for Users

- Always run the latest supported version
- Keep dependencies up to date
- Review configuration for security-sensitive settings
- Report suspicious behavior promptly
